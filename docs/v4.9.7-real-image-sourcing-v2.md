**Full fallback chain (in order):**
1. Library reuse — check `image_library` for matching tags, skip if `times_used >= max_reuse`
2. Google CSE crawl — search, domain check, size filter, download, AI validate, store to library
3. Existing `findRealImage()` — Flickr, Wikimedia, Unsplash, Pexels, Pixabay
4. Branded card (SVG with Inter font, article headline)
5. Sponsor card (if `sponsor_card_enabled`)
6. AI image generation
7. None (article publishes without image)

### Phase 2 — AI Validation ✅

`validateImageRelevance()` calls the LLM with the image URL and article context (headline + tags). Returns a structured JSON response:

```json
{
  "relevant": true,
  "matches_entities": true,
  "safe": true,
  "confidence": 0.87,
  "description": "Photo of the US Capitol building during a press conference"
}
```

**Rejection rules:**
- `relevant: false` → skip immediately
- `safe: false` → skip immediately (no unsafe content)
- `matches_entities: false` AND `confidence > 0.7` → skip (high-confidence wrong subject)

Validation result JSON is stored in the `image_library.validation_result` column for auditing.

## Section B: Google CSE Setup Instructions

To activate the Google CSE image crawler, you need two credentials from Google:

### Step 1 — Create a Custom Search Engine

1. Go to [https://programmablesearchengine.google.com/](https://programmablesearchengine.google.com/)
2. Click **Add** to create a new search engine
3. Under "Sites to search", enter `*` (search the entire web)
4. Enable **Image Search** in the search engine settings
5. Copy the **Search engine ID** (looks like `abc123:xyz456`) — this is your `CX` value

### Step 2 — Get a Custom Search API Key

1. Go to [https://console.cloud.google.com/](https://console.cloud.google.com/)
2. Create a new project or select an existing one
3. Enable the **Custom Search API** for your project
4. Go to **Credentials** → **Create credentials** → **API key**
5. Copy the API key

**Free tier:** 100 queries/day. Paid tier: $5 per 1,000 queries.

### Step 3 — Configure in Hambry Admin

1. Go to **Admin → Settings → Images** (or Wizard Screen 3)
2. Set **Google CSE API Key** to your API key
3. Set **Google CSE Search Engine ID** to your CX value
4. Toggle **Enable Google CSE Image Crawler** to ON
5. Optionally enable **AI Image Validation** (uses LLM credits per image)
6. Set **Min Width** (default 400px) and **Min Height** (default 300px)
7. Set **Max Reuse Count** (default 3 — how many articles can share the same library image)

## Section C: Known Issues and Limitations

| Issue | Severity | Notes |
|---|---|---|
| Google CSE free tier is 100 queries/day | Low | At 50 articles/day, each article uses 1 query = 50/day. Well within free tier. Paid tier available if volume increases. |
| AI validation uses LLM credits per image | Low | Disable `image_validation_enabled` if credits are a concern. The domain whitelist/blacklist provides a first layer of quality control without LLM calls. |
| pHash deduplication is simplified | Low | The current implementation uses a lightweight hash of image bytes (not a true perceptual hash). True pHash requires the `sharp` library's pixel data API. Functional for basic dedup, may miss near-duplicate images. |
| Domain whitelist is conservative | Low | Only 12 whitelisted domains by default. Use Admin → Image Sources → Domain Manager to add more trusted domains as you discover good sources. |
| `image_library` grows indefinitely | Low | No automatic pruning. Add a manual cleanup step in Admin → Image Sources → Image Library if the library grows large. A future version will add age-based pruning. |
| Google CSE may return paywalled images | Low | Even whitelisted news domains (Reuters, AP) sometimes serve images behind paywalls. The size filter and AI validation catch most of these, but some may slip through. |
